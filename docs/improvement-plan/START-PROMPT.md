# 実装セッション開始用プロンプト

以下を新しいセッションの最初のメッセージとして貼り付ける。

---

terminal-board の改修を行います。`docs/improvement-plan/` に監査済みの改修計画一式があります。

進め方:

1. まず `docs/improvement-plan/00-overview.md` を読んで全体像・実装ルール・フェーズ順序を把握してください。
2. フェーズ 0(WIP差分のコミット)から開始し、`01-security.md` → `02-quick-fixes.md` → `03-messaging-reliability.md` → `04-client-performance.md` → `05-aux-safety-refactor.md` → `06-ux-features.md` → `07-appendix-small-fixes.md` の順に実装してください(3 と 4 は順不同可)。
3. 各ドキュメントの行番号は監査時点(コミット e46eb92)の参考値です。必ず記載のシンボル名で grep して現在のコードを確認し、**コードとドキュメントが食い違う場合はコードを正として意図を汲んで適応**してください。
4. 1タスク = 1コミット。各タスクの「受け入れ基準」を検証してからコミットし、フェーズ完了ごとに `npx tsc --noEmit` と `npm run build` が通ることを確認してください。
5. 質問せず自律的に進めて構いません。ただし、ドキュメントに書かれていない破壊的変更(依存の大規模更新、ファイル削除、保存データの非互換変更)が必要になった場合だけ手を止めて報告してください。
6. キーボードナビゲーション機能(ショートカットでのターミナル巡回・コマンドパレット)は意図的にスコープ外です。提案も実装もしないでください。
7. 全フェーズ完了後、実施内容・スキップした項目(あれば理由)・受け入れ基準の検証結果を一覧にして報告してください。

まずフェーズ 0 と 1 から着手してください。
